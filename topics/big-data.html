01 · Introduction

Why Big Data Processing?

Modern companies generate massive amounts of data every day. Traditional tools cannot handle this scale — big data processing systems were built to fill that gap.

📱

Billions of posts, likes, and shares per day
💳

Millions of payments processed per second
🌡️
IoT Sensor Data

🖱️
Website Clickstreams
Every page view and user interaction logged
🎬

Play, pause, seek — tracked for every viewer
🎬 Real Example — Netflix
Netflix collects user activity (what users watch, search history, pause/play events, device info) with millions of events per second. This data drives movie recommendations, user behavior analysis, and streaming performance improvements. This is exactly where big data processing systems are essential.

05 · Apache Hadoop

Apache Hadoop

Apache Hadoop is one of the earliest and most widely used big data processing frameworks. It processes massive datasets using clusters of commodity computers working in parallel — instead of one expensive, powerful machine.

📦 Scale Example — 100 TB of Data
Without Hadoop: One machine would take days or weeks.
With Hadoop: Hundreds of machines process data in parallel, dramatically reducing time.

✅Hadoop Characteristics

🖥️
Distributed Processing
Data processing occurs across multiple machines in a cluster simultaneously.
📈
Scalability
Clusters expand by adding machines: 10 → 100 → 1000 nodes.
🛡️
Fault Tolerance
If one machine fails, Hadoop recovers automatically using data replication.
💰
Cost Efficiency
Runs on commodity hardware — no need for expensive dedicated servers.

🗂️Common Hadoop Ecosystem Frameworks

FrameworkPurpose
Apache HiveSQL queries on Hadoop data
Apache PigData transformation using dataflow scripts
Apache HBaseNoSQL database on top of HDFS
Apache SparkFast distributed in-memory processing
📱 Real Industry Example — Social Media
A social media company processes billions of posts. User posts → stored in HDFS → MapReduce analyzes posts → output shows trending topics and user engagement patterns.
06 · HDFS

Hadoop Distributed File System (HDFS)

HDFS is the storage layer of Hadoop. It distributes large files across multiple nodes in a cluster, enabling high throughput and fault-tolerant storage.

⚙️How HDFS Works

Large files are split into smaller blocks and stored across different nodes. Each block is replicated for fault tolerance.

1 TB File
→
Split into 128 MB blocks
→
Blocks distributed across nodes
→
Replicas stored on backup nodes
BlockPrimary NodeReplica Node
Block 1Node ANode D
Block 2Node BNode E
Block 3Node CNode F
✅ HDFS Advantages
High fault tolerance (automatic failover via replicas) · Distributed storage across many nodes · High throughput for large sequential reads

🧩YARN — Resource Management

Yet Another Resource Negotiator (YARN) is the resource management layer in Hadoop. It acts as the cluster manager, deciding which machine processes which task and how CPU and memory are allocated.

🧠
CPU Management
Allocates processing cores to active jobs across the cluster.
💾
Memory Allocation
Controls RAM usage per task to prevent resource conflicts.
📅
Job Scheduling
Queues and prioritizes jobs across a cluster of hundreds of machines.
07 · MapReduce

Hadoop MapReduce

MapReduce is the core processing model used in Hadoop. It breaks large data problems into two distinct phases for massive parallel processing.

🗺️
Map Phase
Data is split into smaller pieces and processed in parallel across all nodes.
SplitParallelKey-Value
🔀
Shuffle & Sort
Intermediate outputs are grouped by key and transferred to reducer nodes.
GroupSortTransfer
📊
Reduce Phase
Results from map tasks are aggregated to produce the final output.
AggregateSummarizeOutput

📁 Website Log Analysis
Website logs (user visits, pages, timestamps) → stored in HDFS → MapReduce processes logs → output shows most visited pages and traffic statistics.
08 · Apache Spark

Apache Spark

Apache Spark is a powerful open-source framework for large-scale data processing. It performs processing in memory, making it significantly faster than Hadoop MapReduce for many workloads.

⚡ Speed Comparison — 1 TB Website Logs
Hadoop MapReduce: Processing may take hours (disk I/O at every step).
Apache Spark: Processing can finish much faster — intermediate data stays in RAM.

✅Spark Characteristics

🧠
In-Memory Processing
Intermediate data stored in RAM, not disk — dramatically faster iteration.
🌐
Distributed
Processes data across many machines simultaneously.
🐍
Multi-Language
Supports Python (PySpark), Java, Scala, and SQL.
🔄
Batch + Streaming
Handles batch, real-time streaming, and ML workloads.
🛡️
Fault Tolerant
Automatically recovers data using RDDs if a node fails.

🏗️Spark Cluster Architecture

09 · Amazon EMR

Amazon EMR (Elastic MapReduce)

Amazon EMR is the AWS managed service for running big data frameworks — without manually provisioning and configuring infrastructure. It supports Spark, Hadoop, Hive, Presto, and HBase.

☁️ Key Benefit
Instead of manually installing Spark on many machines, engineers launch an Amazon EMR cluster and AWS automatically sets up the entire environment in minutes.

⭐EMR Key Features

⚙️
Managed Platform
AWS handles infrastructure setup, patching, and configuration.
📈
Scalable
Scale from a few nodes to hundreds with auto-scaling support.
💰
Cost Efficient
Clusters can be started and stopped as needed — pay only for what you use.
🔗
AWS Integration
Native integration with S3, Glue, Redshift, and IAM.

🖧EMR Node Types

Node TypeFunctionStores Data?
Main NodeManages the cluster, coordinates resource allocationNo
Core NodesRun tasks and store data in HDFSYes
Task NodesExecute processing tasks only — no data storageNo
🔒 Default Security Note
By default, only the main node is accessible externally. Core and task nodes are not directly accessible from outside the cluster, protecting sensitive data and processing resources.

10 · Apache Hudi

Apache Hudi

Apache Hudi is a framework designed to manage incremental data processing in data lakes. It solves a core problem: traditional data lakes (Parquet/ORC files) are difficult to update efficiently.

❓ Why Hudi is Needed
Without Hudi: Updating a single record requires rewriting the entire dataset file.
With Hudi: Only the changed records are updated — dramatically improving performance.

✅Hudi Characteristics

📊
Incremental Processing
Only new or updated data is processed, not the full dataset.
✏️
Upserts
Supports insert and update operations natively on data lake files.
🕰️
Data Versioning
Maintains a timeline of historical dataset changes for audit and rollback.
⚡
Faster Queries
Optimized file layouts improve query performance on large datasets.

💾Hudi Storage Types

Storage TypeHow Updates WorkBest For
Copy-on-Write (CoW) Updates rewrite the entire file with new data merged in Read-heavy workloads with fewer updates
Merge-on-Read (MoR) Updates stored in delta logs and merged during query time Write-heavy workloads needing fast ingestion

🔑Key Hudi Concepts

ConceptMeaning
DatasetCollection of data files managed by Hudi
CommitA recorded set of data changes applied to a dataset
UpsertInsert a new record or update an existing one atomically
TimelineFull history of all dataset changes, enabling time-travel queries
🚗 Real Industry Example — Ride-Sharing
Trip data → stored in S3 data lake → EMR cluster runs Spark processing → Apache Hudi manages incremental updates → analytics dashboard shows real-time trip statistics.

QUESTION 01

Which tool is an open-source, in-memory SQL query engine that emphasizes faster queries?

✅ Presto is an open-source distributed SQL query engine that performs fast in-memory queries on large datasets across multiple data sources — without moving data.
QUESTION 02 · SELECT TWO

A data engineer is considering the Presto framework for their data analytics workload. What are some benefits of choosing Presto?

Choose exactly two options.


✅ Presto is designed for fast interactive SQL queries using in-memory processing, and it integrates with modern data lake technologies like Apache Hudi for analytics.
QUESTION 03 · SELECT TWO

A company wants to use inexpensive commodity hardware they already own. Which programming frameworks are designed to best operate on that hardware?

Choose exactly two options.


✅ Apache Hadoop is specifically designed to run on low-cost commodity hardware, and Apache Hive runs on top of Hadoop to provide SQL-like querying — both are built for cost-efficient big data processing.
QUESTION 04

A data engineer is planning a machine learning project that involves iterative, multi-stage ML algorithms. Which programming framework best supports these requirements?

✅ Apache Spark MLlib is a distributed machine learning library designed for iterative and multi-stage ML algorithms, using fast in-memory processing that makes repeated model training efficient.
QUESTION 05

A company is migrating on-premises Hadoop workloads to Amazon EMR. What is a key benefit of choosing Amazon EMR?

✅ Amazon EMR can quickly provision scalable clusters in the cloud, allowing organizations to scale faster and handle larger workloads than typical on-premises Hadoop infrastructure — without buying or managing hardware.
QUESTION 06

When launching a cluster, how is access to the core and task nodes handled under default EMR cluster settings?

✅ By default, only the main (master) node is accessible externally. Core and task nodes are protected within the cluster's security group and cannot be directly accessed from outside.
QUESTION 07 · SELECT THREE

Each instance in an Amazon EMR cluster is called a node. What are the EMR cluster node types?

Choose exactly three options.


✅ Amazon EMR has three node types: Main node (manages the cluster), Core node (runs tasks and stores data in HDFS), and Task node (runs processing tasks only, no data storage).
QUESTION 08 · SELECT TWO

What are the two dataset storage types that Apache Hudi supports?

Choose exactly two options.


✅ Apache Hudi supports two storage types: Copy-on-Write (CoW) — updates rewrite the entire file (best for read-heavy workloads); and Merge-on-Read (MoR) — updates stored in delta logs and merged at query time (best for write-heavy workloads).
QUESTION 09

Which processing framework uses Resilient Distributed Datasets (RDDs)?

✅ Apache Spark uses RDDs (Resilient Distributed Datasets) as its core data abstraction — an immutable, fault-tolerant collection of elements distributed across the cluster for parallel processing.
QUESTION 10 · SELECT TWO

A data engineer is considering using the Apache Hadoop framework. What are benefits of the Hadoop framework?

Choose exactly two options.


✅ Hadoop is highly scalable and designed for big data — enabling petabyte-scale storage. It is also open-source and runs on commodity hardware, making it a cost-effective solution for large data processing needs.
