Section 01

What is Data Engineering?

Data Engineering is the discipline of designing, building, and maintaining the infrastructure that moves raw data from its source to the people and systems that need it — clean, reliable, and on time.

💡 Core idea: Data Engineers build the plumbing of an organisation's data system. Without them, analysts and data scientists have no reliable data to work with.

⚙️The Data Engineering Lifecycle

📥
Ingestion
Pull data from APIs, databases, event streams, and files into a central store.
🧹
Transformation
Clean, validate, deduplicate, and reshape data into usable formats.
🗄️
Storage
Persist data in the right store — data lake, warehouse, or database.
🚀
Serving
Expose data through BI tools, APIs, or ML training datasets.

# Simplified Glue PySpark job — transform raw orders
from awsglue.transforms import *
from pyspark.sql.functions import col, to_date

datasource = glueContext.create_dynamic_frame.from_catalog(
    database="raw_db", table_name="orders"
)

df = datasource.toDF() \
    .filter(col("amount") > 0) \
    .withColumn("order_date", to_date("created_at")) \
    .dropDuplicates(["order_id"])

df.write.parquet("s3://curated-bucket/orders/")

🗃️SQL — Query Language for Data

You should be able to write queries that filter, join, and aggregate data. Here is an example you should feel comfortable reading:

-- Find top 10 customers by total spend in the last 30 days
SELECT
    c.customer_id,
    c.name,
    SUM(o.amount)     AS total_spend,
    COUNT(o.order_id) AS order_count
FROM customers c
JOIN orders o ON c.customer_id = o.customer_id
WHERE o.created_at >= CURRENT_DATE - INTERVAL '30 days'
GROUP BY c.customer_id, c.name
ORDER BY total_spend DESC
LIMIT 10;

# MapReduce concept — order count by region

## MAP phase — split input into key/value pairs
"order_id=101, region=US"  → ("US", 1)
"order_id=102, region=EU"  → ("EU", 1)
"order_id=103, region=US"  → ("US", 1)

## REDUCE phase — aggregate by key
("US", [1, 1]) → ("US", 2)
("EU", [1])    → ("EU", 1)

Q 01
Who is primarily responsible for building and maintaining data pipelines?
✅ Data Engineers design, build, and maintain the pipelines that move and transform data. Analysts and Scientists consume what Engineers produce.
Q 02
Which AWS service is the primary choice for a Data Warehouse workload?
✅ Amazon Redshift is AWS's managed columnar data warehouse, optimised for OLAP queries. S3 is a data lake; RDS is for transactional (OLTP) workloads.
Q 03
What does the acronym ETL stand for?
✅ Extract, Transform, Load — the classic data integration pattern. Extract from sources, Transform to clean and shape the data, then Load into a target store.
Q 04
Which AWS service would you use to run a serverless ETL job on data stored in S3?
✅ AWS Glue is a fully managed, serverless ETL service. It can crawl S3, auto-discover schemas, and run Spark-based transformation jobs without managing servers.
Q 05
What is the correct order of stages in a standard ML pipeline?
✅ You always start with raw data, clean it, engineer features, train the model, evaluate it, and finally deploy to production.
Q 06
Which storage format is best for large-scale analytical queries in a data lake?
✅ Apache Parquet is a columnar format with built-in compression — it dramatically reduces data scanned (and cost) for analytical queries compared to row-based CSV.
